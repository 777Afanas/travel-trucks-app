# 🚐 TravelTrucks

Сучасний вебдодаток для оренди кемперів, який дозволяє переглядати доступні будинки на колесах, фільтрувати їх за різними характеристиками, вивчати профілі кемперів із відгуками та миттєво бронювати їх для наступної подорожі. Проєкт реалізовано як фронтенд-додаток у продакшн-стилі з використанням React, Vite, Redux Toolkit та стандартних підходів до управління станом.

## 📝 Про проєкт

TravelTrucks — це інтерактивна платформа для оренди кемперів, створена для допомоги користувачам при пошуку та бронюванні транспортних засобів для подорожей.

**Додаток дозволяє:**

- Переглядати повний каталог доступних кемперів.
- Динамічно фільтрувати доступні транспортні засоби за локацією, типом кузова та конкретними зручностями (наявність кондиціонера, кухні тощо).
- Довантажувати картки транспортних засобів на сторінці каталогу з урахуванням обраних фільтрів (кнопка "Load More").
- Переглядати детальний опис обраного транспортного засобу, в т.ч. — фотогалереї, повні технічні характеристики та наявні відгуки користувачів.
- Здійснювати бронювання кемперу шляхом заповнення форми на сторінці обраного кемпера та отримувати повідомлення про вдале бронювання (нотифікацію).

---

## ⚡ Основні функції

### 1. Каталог та динамічний пошук

- **Розумна фільтрація:** Пошук за локацією (текстове поле), типом кузова (один варіант) та кількома критеріями обладнання одночасно (кондиціонер, кухня, ТВ тощо).
- **Скидання фільтрів:** Можливість скинути попередні результати пошуку, щоб забезпечити актуальність та точність відображуваних даних відповідно до нових критеріїв.
- **Пагінація "Load More":** Можливість довантажувати додаткові картки кемперів без втрати поточних налаштувань фільтрації.

### 2. Детальне вивчення профілю та відгуків

- **Повні характеристики:** Відображення детальних експлуатаційних параметрів (трансмісія, двигун, розміри, бак, витрата палива).
- **Зворотний зв'язок:** Відображення відгуків клієнтів із використанням 5-зіркової шкали оцінювання.

### 3. Здійснення бронювання

- **Форма бронювання:** Зручна форма для бронювання обраного кемпера, яка надсилає миттєве нотифікаційне сповіщення про успішне відправлення запиту.

---

## 🛠️ Технологічний стек

- **Фронтенд:** React, Vite, React Router (Маршрутизація)
- **Управління станом:** Redux Toolkit (Глобальний стан для кемперів, фільтрів та обраного)
- **Запити до API:** Axios
- **Стилізація:** CSS Модулі
- **Сповіщення:** iziToast / react-toastify

Проєкт інтегрується з готовим бекендом API: `https://66b1f8e71ca8ad33d4f5f63e.mockapi.io/campers`

**Використані ендпоінти:**

- `GET /campers` — Отримання всіх оголошень (в т.ч. з визначеними параметрами фільтрації).
- `GET /campers/:id` — Отримання детальних даних про конкретний кемпер за його унікальним ID.

## 📁 Структура проєкту

```plaintext
TRAVEL-TRUCKS-APP/
├── public/          # Статичні файли (іконки, SVG-спрайти)
├── src/
│   ├── components/  # Перевикористовувані компоненти (CamperCard, Button, Loader тощо)
│   ├── layouts/     # Компоненти обгорток (напр., Layout зі спільним Header/Navigation)
│   ├── pages/       # Сторінки додатку (Home, Catalog, CamperDetails)
│   ├── redux/       # Налаштування Redux (стор, слайси та асинхронні thunks для API)
│   ├── App.jsx      # Головний компонент додатку та конфігурація маршрутів (React Router)
│   ├── index.css    # Глобальні стилі додатку
│   └── main.jsx     # Точка входу в React-додаток
├── index.html       # Головний HTML-шаблон
├── package.json     # Залежності проєкту та npm-скрипти
└── vite.config.js   # Конфігурація збірника Vite
```
